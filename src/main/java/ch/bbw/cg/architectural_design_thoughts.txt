There are some questions I asked myself during the process:
1. How do I define the game board and its cells?
    --> for standard rules I guess it works to think of the board as a whole and just take a two-dimensional boolean or integer array with rows and columns defined as integer. The position is sufficient. Even with multiple cell states the two-dimensional array should be enough, when it is containing integers instead of booleans.

2. Problem was that the two-dimensional boolean array needs 2 states, because I change the cells directly in the array one by one. So every change directly impacts the cell's neighbours. There should be the whole board first processed before calculating the next generation.
    --> Either I copy the whole board into a new array and process it there or look at the cells as objects and process their states directly.
    --> I think the second approach is better, because it is more flexible for later changes.
